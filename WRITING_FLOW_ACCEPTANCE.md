# 一键写作流程验收记录

## 优化前步骤

1. 桌面启动脚本打开首页 `http://localhost:3000`。
2. 用户需要自行进入 Studio、选择内容类型、点击创建草稿，再进入编辑页面。
3. 未登录用户登录成功后固定进入 `/studio`，不会返回原本的写作目标。
4. 标题、Slug、摘要、标签、精选和项目字段都排在正文前或同一编辑区域内。

## 优化后步骤

1. 桌面“开始写作”快捷方式启动或复用本地服务，并打开 `/studio/new`。
2. `/studio/new` 复用当前用户仍为空的默认心得草稿；没有可复用草稿时才创建一条。
3. 页面直接进入标题、内容类型和 Tiptap 正文编辑器；摘要、标签、Slug、封面地址、精选和项目设置折叠到“更多设置”。
4. 标题在打开时聚焦并选中默认标题；按 Tab 会进入正文；`Ctrl/Cmd+S` 可立即保存。

## 登录回跳结果

- proxy 对未登录的 `/studio/new` 跳转到 `/login?next=/studio/new`。
- 登录页和登录 Server Action 都通过 `getSafeStudioRedirect()` 限制目标：只允许单斜杠开头的 `/studio` 路径，拒绝 `//`、反斜杠和站外地址。
- 已登录管理员访问登录页会直接回到合法的 `next` 路径；非管理员登录后仍会被拒绝，Studio layout 与 RLS 不变。

## 自动聚焦结果

- 已实现：写作页传入 `autoFocusTitle`，标题 input 在客户端挂载后执行 `focus()` 和 `select()`。
- 已实现：标题输入框按 Tab 时直接调用 Tiptap `focus("start")`；编辑器尚未就绪时会在就绪后补焦点。
- 未做浏览器人工验收：当前执行环境没有可用的管理员登录凭据和交互式浏览器会话，不能如实声明已完成真实视觉验证。

## 空草稿防重复结果

- 已实现：只复用当前管理员创建的、未删除的默认心得草稿，且要求标题、正文、摘要、封面、标签、精选状态和 Tiptap JSON 都保持空白。
- 已实现：有正文或任意设置的草稿不会被当作空草稿复用；刷新页面会继续使用同一条空草稿，不会再次插入。
- 边界说明：数据库结构未改动，因此无法为两个完全并发的首次请求提供数据库级唯一约束；正常连续打开、刷新和桌面快捷方式重复启动均先查询再复用。

## 自动保存结果

- 已保留阶段 4 的 1200ms debounce 自动保存、保存状态提示、localStorage 恢复和离开页面未保存提醒。
- 已新增 `Ctrl/Cmd+S` 立即保存。
- 未做真实登录后的数据库写入验收：当前环境没有可使用的管理员账号，不能伪造自动保存和刷新后的持久化结果。

## 实际测试结果

- `npm run lint`：通过。
- `npm run typecheck`：通过。
- `npm run build`：通过（受限沙盒内会出现 Next.js 子进程 `EPERM`，已在批准环境复跑通过）。
- 已人工审查 Windows 脚本：只使用脚本所在项目目录、系统 Program Files 路径和 `localhost:3000/studio/new`；不包含密码、Supabase 密钥、管理员 UID、用户绝对路径或 `node_modules`。
- 待管理员手工验收：退出登录 → `/studio/new` → 登录回跳 → 标题焦点 → 输入与自动保存 → 刷新 → 再次双击桌面快捷方式。
