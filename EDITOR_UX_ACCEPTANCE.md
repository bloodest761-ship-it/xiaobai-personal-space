# 编辑器工具栏与正文排版验收记录

## 优化前问题

- 所有工具栏按钮平铺在一个可换行容器中，缺少功能分组、提示和移动端溢出策略。
- 工具栏不是 sticky；正文编辑样式在 `globals.css`，公开渲染则主要由 `RichContent.tsx` 的内联类定义，层级不一致。
- 编辑器正文、列表、引用、代码和自定义节点的视觉规则较少；公开端的自定义块与编辑端辨识度不同。
- 标题输入框使用普通表单样式，正文没有空内容提示。

## 工具栏结构

- 新增 `EditorToolbar.tsx`，按文本类型、文字格式、结构、插入、历史五组排列，并使用克制的分隔线。
- 每个按钮都有可理解的文字、`aria-label` 和浏览器原生简洁提示；激活状态同时有背景、边框和字重变化，不只依赖颜色。
- 撤销、重做根据 Tiptap `can()` 状态禁用；工具栏按下时保留当前选区，命令执行后恢复编辑器焦点。
- 工具栏在桌面端使用 sticky 定位；手机端单行横向滚动，隐藏滚动条但保留触控滚动和足够点击面积。

## 编辑区排版

- 新增共享 `content-typography` 规则：最大宽度 48rem（768px）、中文正文 17px / 1.8 行高、H2/H3、列表、引用、代码、链接、分割线、图片及图片说明统一处理。
- 编辑器增加清晰的光标色、选区样式和“从这里开始记录你的想法……”空正文提示。
- 标题改为大字号、底边线的写作式输入区，并保留自然 placeholder；标题 Tab 会进入正文。

## 公开端一致性

- `RichContent.tsx` 改用同一 `content-typography` 类，删除公开端对标题、列表、引用、代码和行内代码的独立排版分叉。
- 提示块、认知变化块、项目概览块使用共享内容块风格；公开端项目概览会隐藏空字段，仓库链接可访问并在新标签页打开。
- 编辑器自定义原子节点保持原有 JSON 数据结构，仅调整视觉样式和选中状态。

## 手机端测试

- 代码审查：工具栏设为横向滚动而非换行，按钮最小高度 40px，不会撑破 375px 宽度。
- 代码审查：正文在窄屏降为 16px，缩小内边距，代码块保持横向滚动。
- 未进行真实设备或交互式浏览器 375px 验收。

## 键盘测试

- 代码审查：Tiptap StarterKit 继续提供 Ctrl+B、Ctrl+I、Ctrl+Z、Ctrl+Shift+Z/Ctrl+Y；编辑器自动保存逻辑继续通过 1200ms debounce 运行。
- 代码审查：`Ctrl/Cmd+S` 由写作表单拦截并立即保存；标题 Tab 会把焦点移入 Tiptap 正文。
- 未使用管理员登录会话真实执行键盘操作。

## 性能检查

- 未增加任何依赖；工具栏状态由 `useEditorState` 在独立客户端组件内更新，避免为按钮状态重渲染整个写作表单。
- Tiptap、工具栏和图片上传组件仍只由 `/studio/new` 与编辑页加载；公开首页和列表不会引入编辑器代码。
- 自动保存仍是 debounce；工具栏不进行按键级重型计算。
