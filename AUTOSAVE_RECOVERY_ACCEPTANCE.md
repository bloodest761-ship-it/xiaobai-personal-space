# 自动保存、失败重试与内容防丢失验收

## 原有自动保存流程

`EntryEditorForm` 在字段变更后标记 `unsaved`，将 payload 写入 localStorage，并在 1.2 秒后调用 `saveEditorEntryAction`。Ctrl+S、手动保存和发布各自直接调用保存相关函数。服务端已有 `expected_updated_at` 乐观并发校验。

## 发现的问题

- 强制保存可能并发发出，旧请求返回时会干扰较新内容的状态。
- 本地草稿没有格式版本、同步标记或服务器时间比较，存在草稿即提示恢复。
- 保存失败没有统一重试、联网恢复和错误分类；发布未排空客户端保存流程。
- 离开页面时只弹浏览器提示，未明确先写入最新本地快照。

## 新的保存状态

- `未保存`
- `正在保存…`
- `已保存`（显示最近服务器保存时间）
- `保存失败`
- `离线，已在本地备份`
- `正在恢复…`

## 保存状态机

```text
编辑变化 → 未保存 → 1.2s debounce → 正在保存 → 已保存
                    │                  │
                    │                  └─ 继续编辑 → 未保存（仅保存最新快照）
                    └─ 离线/失败 → 本地备份 → 离线或保存失败 → 重试/联网后一次重试
```

## 竞态处理方式

- 每次变更递增 `revision`，并更新唯一的完整快照 ref。
- 保存采用单飞队列：同一时刻只发送一个 Server Action；保存期间的新输入只标记队列，前一个请求结束后再保存最新快照。
- 只有已完成请求对应的 revision 仍是最新版本，才会标记“已保存”；旧请求不会把较新的内容误标为已保存。
- 服务端保留原有 `updated_at` 乐观并发检查，跨标签页/跨客户端的旧版本仍会被拒绝而不是覆盖。

## 本地备份结构

- key：`xiaobai-draft:<entry-id>`，新建写作页在真实 entry ID 创建后才渲染，因此直接使用正式 key。
- 包含格式版本、entry ID、状态、完整编辑快照、客户端修改时间、服务器更新时间、同步时间和标签页 ID。
- 完整快照包含标题、slug、类型、摘要、封面、正文 JSON/文本、标签、精选设置、metadata 与服务端版本。
- 不保存 Supabase 密钥、登录 Token 或图片二进制；图片仅作为正文 JSON 中的 URL/节点保存。每篇文章只覆盖自己的一个最新备份。

## 恢复判断逻辑

- 仅当备份格式有效、尚未同步且客户端修改时间晚于当前服务器内容时，显示恢复提示。
- 用户可恢复本地内容、使用服务器内容或稍后决定；不会自动覆盖服务器内容。
- 损坏/旧格式或 localStorage 不可用时安全忽略并给出简短提示；同一次打开不会反复提示。

## 失败重试流程

- 保存失败保留编辑器内容并写入未同步备份，显示中文错误和“重试保存”。
- Ctrl+S、手动保存、自动保存、失败重试都复用同一个保存队列；重试永远读取当前最新快照。
- 网络失败在 `online` 事件后仅自动尝试一次；不会无限重试。登录、权限、网络、服务器和未知问题使用不同用户提示。

## 发布前保存流程

```text
发布 → 清除 debounce → 保存最新完整快照 → 成功后调用发布 Action → 更新缓存与同步备份
                         └─ 失败：阻止发布、保留备份、显示“最新内容尚未保存，无法发布”
```

## 离开页面保护

- 有未同步变更时使用浏览器标准 `beforeunload` 提示；已保存不会提示。
- 页面隐藏、刷新或关闭前会同步写入 localStorage，不依赖 beforeunload 中完成网络请求。
- 站内正常导航不被自定义弹窗阻断。软删除成功后，列表页会清理对应文章的本地备份。

## 中文输入法处理

- Tiptap 在 `compositionstart` 到 `compositionend` 期间不触发保存快照更新。
- `compositionend` 后才提交完整 JSON 并进入 debounce，不调用 `setContent` 或重新初始化编辑器。

## 代码级验证

- `revision`、单飞队列、失败重试、服务器版本更新、备份格式校验与恢复比较已完成代码检查。
- 图片 alt/caption 通过既有 `RichTextEditor.onChange` 进入同一完整快照；项目 metadata 也使用同一更新入口。
- 未引入新的依赖、状态管理库、数据库变更或 RLS 修改。

## 浏览器实际验证

未验证：当前没有可登录的浏览器会话与测试账号，未执行真实输入、网络断开、刷新恢复、发布、关闭页面、移动端或多标签页操作。

## 未验证内容

- 真实 Server Action 网络错误的分类与联网后自动重试。
- 真实管理员会话中的发布前保存、刷新恢复与 375px 视觉效果。
- 浏览器中文输入法、多标签页 storage 事件和软删除后的客户端备份清理。

## 后续风险

- 跨设备/跨标签页只提供保存冲突保护和编辑提示，不提供冲突合并。
- localStorage 被浏览器禁用或存储空间耗尽时，应用安全降级，但无法提供本地恢复能力。
